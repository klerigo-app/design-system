# Design Sync Notes

## Known warnings (triaged)

- `[FONT_MISSING] "Trebuchet MS"` — this is a fallback in the `--font-display` stack (`'Baloo 2', 'Trebuchet MS', sans-serif'`), not the intended font. Baloo 2 ships its own `@font-face` via `@fontsource/baloo-2` and is what actually renders. Trebuchet MS is a Windows system-font fallback for the rare case Baloo 2 fails to load — no action needed.

## Config decisions

- No `cfg.cssEntry` set. The package ships CSS as two separate dist files (`dist/index.css` = Tailwind/component styles, `dist/fonts.css` = `@fontsource` imports + `--font-*` custom properties) that are NOT concatenated by the build (`package.json`'s `build` script just copies `fonts.css` into `dist/` — it's a separate export, `./fonts.css`). Storybook's `preview.tsx` imports both, so relying on `[CSS_FROM_STORYBOOK]` (scraping the compiled iframe CSS) captures the full closure correctly, whereas setting `cssEntry` to just `dist/index.css` caused `[TOKENS_MISSING]` for `--font-mono`/`--font-body`/`--font-display`. Leave `cssEntry` unset.
- `titleMap: {"Overview": null}` — `Tokens/Overview` is a token-showcase story (color ramps, radii), not a real component export; excluded.
- `overrides.Modal: {cardMode: "single", primaryStory: "TypeToConfirmDelete"}` — Modal renders via a fixed-position overlay/portal (`[GRID_OVERFLOW]`/`[RENDER_THIN]` in validate), so no grid layout can present its stories side-by-side. `TypeToConfirmDelete` was picked as the primary story since it demonstrates the type-to-confirm gating that differentiates this Modal from a generic one.
- `overrides.Modal.skip: ["core-modal--wide-with-side-by-side-fields"]` (added 2026-07-22, 6th sync) — the `WideWithSideBySideFields` story (new in #12, demos the `size: 'lg'` prop = `max-w-xl` with two side-by-side `TextInput`s) renders `isOpen` with **no `Abrir modal` trigger button**, so its storybook story root is genuinely empty (the Modal portals to `document.body`, painting outside the root's bounding box). Compare marks it `sb-error` ("no storybook root content") — un-gradeable by the oracle, unlike the 6 variant stories which have a trigger button anchoring their (gated) reference. `size: 'lg'` is a benign width class verified by reading `Modal.tsx` (`modalPanelStyles` cva: `md→max-w-md`, `lg→max-w-xl`). Skipped since it can't be compared and adds no fidelity beyond the already-verified TextInput + Modal-variant coverage. This appears only at `--max-stories 7` (default cap is 6); the skip makes it deterministic so a future cap-raise doesn't re-surface the "error".
- `readmeHeader: ".design-sync/conventions.md"` — authored conventions covering the Tailwind-preset styling idiom (real class names like `bg-coral-500`, `shadow-lift-coral`, `rounded-card`), no-provider-needed setup, and an idiomatic composition example. Validated every named class/prop against the compiled `_ds_bundle.css` and component source before shipping.
- `storyImports.shim: ["useToast"]` (added 2026-07-13, 4th sync) — Toast's `Interactive` story wraps its own content in `<ToastProvider>` (from `./ToastProvider`) and calls `useToast` (from `./useToast`). The preview-import policy only shims relative imports whose basename is in the bundle's **component** export set; `ToastProvider` (capitalized) shimmed to the global, but `useToast` (lowercase hook) did NOT — it bundled from source, pulling in a **second** `ToastContext = createContext(null)`. So the shipped provider fed context A while the source hook read context B → `useToast must be used within a <ToastProvider>` at render. Forcing `useToast` to shim makes the facade's `export * from window.GLOBAL` resolve it to the bundle's single context. Fix verified: `grep createContext ds-bundle/_preview/Toast.js` is now empty and all 5 Toast stories (incl. Interactive) grade match. **If another lowercase hook-style export ever gets used inside a story's own provider, it needs the same `storyImports.shim` entry.**
